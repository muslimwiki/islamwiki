## 🎯 **Release Overview**

Version 0.0.2.8 resolves a critical authentication issue that was preventing non-admin users from accessing the `/wiki` pages, while also implementing a comprehensive enhanced error handling system for better debugging and troubleshooting.

## 🔐 **Critical Authentication Fix**

### **Issue Resolved**
- **Non-admin users could not access `/wiki` pages** - This was causing a 500 error for regular users
- **Missing user role information** - Authentication service was not returning proper user role data
- **Session data retrieval failure** - User session data was not being properly retrieved

### **Root Cause**
The authentication service (`AmanSecurity`) was trying to access a `role` field in user data that didn't exist in the database. The `users` table has `islamic_role` and `is_admin` columns, but the code was expecting a computed `role` field.

### **Solution Implemented**
- **Enhanced `AmanSecurity::user()` method** - Now properly computes and returns the `role` field based on `is_admin` status
- **Fixed session data retrieval** - User authentication data now includes the computed role information
- **Improved error handling** - Better debugging information when authentication issues occur

### **Result**
✅ **All user types can now access `/wiki` pages:**
- Guest users (not logged in)
- Non-admin users (logged in)
- Admin users (logged in)

✅ **Quick action buttons are now visible and functional for all user types**
